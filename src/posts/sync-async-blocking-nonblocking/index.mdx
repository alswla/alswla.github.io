## 🗂️ 목차

저번 면접 때, 면접 질문으로 Blocking과 Non-blocking에 대해 질문을 받았고, 이에 대한 저의 답변이 무언가 잘못되었다는 것을 느꼈습니다. 우선 질문을 받고도 ‘_Sync/Async가 아니라 Blocking/Non-Blocking은 뭐지?’_ 라는 생각에서부터 제가 무언가를 놓치고 있다는 것을 느끼게 되었고, 해당 면접 이후 바로 이에 대해 공부하고 글을 쓰게 되었습니다.

## Sync/Async 그리고 Blocking/Non-Blocking

블로킹과 논블로킹은 나중에 생각하더라고 Sync/Async 즉 동기/비동기에 대해서는 모두가 익숙할 것이라고 생각합니다. 특히 자바스크립트, Node.js에서는 비동기 이벤트 처리 환경을 지원하기 때문에 이 비동기에 대해서는 자바스크립트를 공부한다면 정말 익숙한 단어일 겁니다.

그렇다면 블로킹/논블로킹은 뭘까요? 동기/비동기와 유사한 개념이 아닌가요? 이런 의문이 들 수 있습니다. 하지만 동기/비동기와 블로킹/논블로킹은 표현 형태는 비슷해 보일지라도 서로 다른 차원에서의 작업의 수행 방식을 설명하는 개념입니다.

Blocking / Non-blocking 은 호출된 함수가 호출한 함수에게 제어권을 바로 주느냐 안주느냐의 차이이고, Sync / Async 는 호출된 함수의 종료를 호출한 함수가 처리하느냐, 호출된 함수가 처리하느냐의 차이입니다.

좀 더 간결하게 말하면, **동기/비동기는 요청한 작업에 대해 완료 여부를 신경써서 작업을 순차적으로 수행할지 아닌지에 대한 관점이고, 블로킹/논블로킹은 단어 그대로 현재 작업이 차단되느냐(Block) 아니냐에 대한 관점**입니다.

이처럼 두 개념에 대한 의미 차이는 명확하지만, 프로그래밍에서는 종종 혼용되어 사용하기도 합니다. 대표적으로 자바스크립트의 `setTimeout()` 함수를 많은 책들에서 일반적으로 비동기 함수라고 부릅니다. 하지만 `setTimeout()` 함수는 비동기 함수이면서 동시에 논블로킹 함수입니다. 즉, 비동기 + 논블로킹 함수인 것입니다.

## Sync와 Async에 대해 더 자세히 알아보자


Synchronous는 작업 시간을 함께 맞춰서 실행한다라는 뜻으로 해석되며 작업을 맞춰 실행한다는 것은 결국 요청한 작업에 대해 완료 여부에 따라 순차대로 처리한다는 것을 의미합니다.

Asynchronous는 Synchronous에 접두사 A가 붙어 부정하는 형태로 동기와 반대로 요청한 작업에 대해서 완료 여부에 따라 처리되지 않기 때문에 다음 작업을 그대로 수행하게 됩니다.

![Sync/Async 도식](/posts/sync-async-blocking-nonblocking/untitled-4.png)

그렇다면 비동기와 동기의 특징에 대해 좀 더 알아보겠습니다.

비동기를 말할 때 주로 성능과 연관지어 말합니다. 요청한 작업에 대하여 완료 여부를 신경쓰지 않고 자신의 그 다음 작업을 수행하는 것은, I/O 작업이나 네트워크 요청과 같은 시간이 소요되는 작업이 발생할 시 해당 작업들이 끝나기를 기다리지 않고 다른 작업을 동시에 처리하는 멀티 작업을 진행할 수 있기 때문입니다.

![멀티 작업 도식](/posts/sync-async-blocking-nonblocking/untitled-5.png)

예들 들어 데이터베이스를 처리하는 작업에서 대규모 트래픽이 발생할 경우, 하나의 데이터베이스 처리가 끝날 때까지 다른 요청을 처리하지 못하므로 웹 애플리케이션의 성능이 저하될 수 있습니다. 하지만 비동기 방식의 처리가 가능할 경우 해당 데이터베이스를 처리할 동안 다른 작업을 **동시에 처리**할 수 있으므로 대규모 트래픽에서도 안정적으로 동작할 수 있습니다.

## 동시처리는 어떻게 진행될까?


그렇다면 이때 동시 처리는 어떻게 진행되는 걸까요? 동시처리란 말 그대로 두 개 이상의 작업이 동시에 실행되는 것을 의미합니다. 이는 멀티 스레드나 멀티 프로세스와 같은 방식으로 구현할 수 있습니다.

_(스레드와 프로세스에 대한 자세한 포스팅은 프로세스와 스레드 글을 참고해주세요)_

자바스크립트의 경우 싱글스레드이지만 비동기로 작업을 요청 시 브라우저에 내장된 Web API에 작업을 전달하여 콜스택과 작업이 동시에 진행됩니다. 이후 Web API에서 콜백함수를 콜백큐로 전달하면, 콜스택의 실행이 모두 끝난 이후 콜백큐의 작업들이 콜스택으로 넘어가 최종적으로 실행됩니다.

대표적인 비동기 작업의 종류로는 네트워크 통신, 애니메이션 실행, 타이머 등 여러가지가 있는데, 자바스크립트 코드의 경우 처리는 Web API에서 수행되지만, 실제 코드의 실행은 Web API가 아닌 콜스택에서 실행됩니다. _(자세한 내용은 이벤트 루프를 공부해 보세요)_

위에서 **동기와 비동기는 요청한 작업에 대해 완료 여부에 대한 차이**라고 말했고, **비동기의 경우 동시에 다른 작업들이 실행된다는 것을 의미**한다고 말했습니다. 그렇다면 이때 동시처리란, 여러 개의 작업의 요청을 동시에 실행하는 것
